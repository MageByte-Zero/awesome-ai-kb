# 写作指导

总体上按照 **标题 + 排版 + 内容** 的权重来创造文章，内容可以是图文形式，纯代码阅读体验极差，不适合在微信公众号这种碎片时间阅读，通常 5 -7 分钟内可以阅读完的长度。

**标题突出重点 + 第一屏概括全文，让读者知道写的是什么，可以使用思维导图、目录、架构图等 + 图文通俗易懂的方式，不要纯代码 + 文章不要太长，更多是通勤时间阅读，5 - 7 分钟即可**

标题：影响阅读

内容：影响分析

## 标题

直接影响是否会点击阅读，所以标题直接切中要点，突出重点让读者知道你要干嘛，看后读者是否觉得有用感兴趣继续点进来

## 排版

第一眼看上去是否舒服，**第一屏很重要，展示出文章的精华概括，可以是目录、思维导图、架构图、核心点，一目了然。** 吸引人往下读，让读者一眼知道你要分享什么。不要站在程序猿的角度去写文章，站在读者的角度，自己是否会读下去。

当读者觉得对自己有用，现在没时间看也会收藏到公司去看。

## 内容

一篇文章不要太多，说明白一个知识点就可以了。不要使用纯代码方式、代码可以截图。并且写之前使用思维导图列一下要点，串连一遍。使用通俗易懂的图文方式，一篇文章大概 6000 即可。
