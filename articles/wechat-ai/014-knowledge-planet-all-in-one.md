# 腾讯 CodeBuddy 核心设计者说，这才是 AI 编程的正确打开方式

![五月 AI 编程工具集体爆发：Gemini CLI、OpenAI Codex 移动端、Claude Code Remote Agents 封面图](https://magebyte.oss-cn-shenzhen.aliyuncs.com/myself/014-knowledge-planet-all-in-one-cover.png)

## 五月，AI 编程工具集体交卷

五月上半月，AI 编程这条赛道密集发生了几件事，我觉得值得单独说一下。

5 月 6 日，Anthropic 举办了 Code with Claude 2026 大会。这次没有发布新模型，主角是 Claude Code 本身：Remote Agents 正式上线（你可以在手机上远程批准或打断跑在服务器上的 Agent 任务）、Claude Code 对 Pro 和 Max 用户的单次会话时间限制直接翻倍、CI 流水线自动修复 PR 进入测试。Anthropic 顺带公布了一个数字：过去一年 Claude Code 的 API 调用量增长了 17 倍。

5 月 12 日，Google 发布 Gemini CLI。完全开源，在终端里直接调用 Gemini 3 Flash（他们说这个模型比旧版 Gemini 2.5 Pro 快 3 倍），用个人 Google 账号登录免费用，每天 1000 次请求、每分钟 60 次。GitHub Actions 集成也同步进入 beta，可以让 Gemini 帮你做 PR review 和 issue triage。一个字：免费、开源、进终端。

5 月 14 日，OpenAI 把 Codex 搬进了 ChatGPT 手机 App，iOS 和 Android 都有，而且是向所有方案开放，包括免费用户。你可以在手机上批准 Agent 的决策、查看 diff、重定向任务，或者直接终止一个跑偏的执行。桌面端这个月也加了 /goal 指令——给 Codex 一个高层级目标，它会在后台异步跑完再通知你，状态持久化跨 session 保存。

Cursor 这边，v3.3 把云端 Agent 能力做了一次大升级：多 repo 环境支持（一个 Agent 可以同时操作多个仓库，跨仓库推理变更影响），还加了 /multitask 指令让多个子 Agent 并行跑。

四家公司，同一周内密集落地。放到一年前，任何一条单独拿出来都是大新闻。

我观察到一件事：这波更新的共同指向不是"更聪明的 AI"，而是**让 AI 从你的对话框里跑出来，真正接入你的工作流**。Codex 进手机，本质上是让 Agent 可以在你不在电脑前的时候继续工作；Claude Code Remote Agents 和 Cursor 多 repo 环境，本质上是在解决"我怎么管理一堆并行跑的 AI 任务"的问题。

工具的形态在变，但大多数人的使用方式还停在"打开软件，问问题，复制回答"这个层次。

在这个节点，我想聊一件对我来说更重要的事。

## 这个星球里有一个人，你在公开渠道很难见到

**林强**——腾讯云开发者 AI 产品首席架构师、腾讯云架构师名人堂专家，腾讯云 AI 代码助手 CodeBuddy 的技术负责人与核心设计者。

我说"公开渠道很难见到"，不是在拔高他的身价。是因为他做的这件事，决定了他的经验有某种稀缺性：当所有用 AI 编程工具的人都在讨论"用什么工具""哪个 Prompt 效果好"的时候，他要解决的问题是——

怎么让几万工程师在真实企业环境里用起来？代码不能出去，知识库在内网，合规要求一堆，怎么让 AI 的能力真正落下来？这个功能加下去，会不会把代码质量搞乱？这个工作流设计，在团队里推广的时候会遇到什么阻力？

这类问题，不是用过 Cursor 或者 Claude Code 几个月就能回答的。他参与了 CodeBuddy 从产品设计到企业落地的全过程，踩过的坑来自真实的企业部署场景，而不是个人开发环境的调优。

你在公开博客和教程里，很难找到这个密度的第一手经验。这是我把他请进星球的原因，也是我觉得星球里最稀缺的那部分内容。

第二位嘉宾是**李运华**——前阿里 P9。P9 在阿里的技术体系里是什么位置，做过大厂的工程师应该清楚。他参与过阿里多个核心系统的设计，在系统设计和大规模架构上的实战经验，是星球里另一块我自己没法替代的内容。

我自己是《Redis 高手心法》的作者，电子工业出版社出版，InfoQ 签约作者，在一线大厂做后端架构，也是腾讯云架构师同盟名人堂专家。但光我一个人，视角终究有边界。

三个背景不同的人在同一个星球里，讲的是同一件事的不同层次——这是我建这个星球的设计思路，不是拼凑。

## 两类人，各走各的路

这个星球现在服务两类差别不小的读者，他们的入口不一样。

**如果你是 IT 小白**，我说的不是"完全没用过电脑"，而是产品经理、运营、内容创作者、销售——会用 AI 问问题，但工作流层面还没有真的动起来。

你的入口是《AI 超能工具箱：普通人也能玩转 Skill、Plugin 与 MCP》，31 讲，专门为不写代码的人设计。

这套内容想解决一件事：大多数非技术岗的人，被"这是程序员的事"这个误解挡在门外。但用好 AI 工具，根本不需要会写代码。

具体讲什么？举几个点：

**claude-mem**，让 AI 永远记住你。你的职业背景、写作风格、习惯用语，存一次，以后每次打开都知道你是谁，不用每次重新介绍。对每天要写东西的人来说，这一个工具就值回来了。

**doc-coauthoring**，周报、方案、述职报告、客户提案，AI 起草，你审稿。重点不是"生成文字"——那个大家都会——而是怎么让 AI 匹配你自己的口吻，不是一堆你再删到一半的 AI 腔文字。

**chrome-devtools-mcp**，AI 帮你点网页、填表单、截图、抓内容。那些每周要重复几十遍的无聊操作，真的可以不用手动做了。

**skill-creator**，把你总结的工作方法写成 Skill，让 AI 以后每次都按你的方式做事。你的方法论，不再只存在你脑子里。

我身边有个做运营的朋友，用了 6 个月，现在接的副业单子比全职工资还高，用的就是这些工具。她一行代码都不会写。

**如果你是 IT 高手**，后端工程师、架构师、技术 leader，你的入口是《AI 编程实战：用 Claude Code 重塑你的开发工作流》，41 讲，6 个模块，加 3 讲加餐。

这套内容不打算只讲"让 AI 写代码"——那一步大多数工程师已经会了。真正麻烦的是这些问题：这个任务该不该交给 AI？需要给它多少上下文？它改完之后怎么验证？哪些动作必须加权限边界？团队里十个人都开始用 AI 后，规范、成本、合规、审计怎么管？

41 讲里有一条从浅到深的主线：从认知层（LLM、Agent、Claude Code、Cursor 是什么关系）到工具配置（CLAUDE.md、Hooks、Permissions 各解决什么问题），再到 MCP 和 Skill 改造工作流，再到企业落地的安全、脱敏、RAG 知识库、成本监控，最后到架构师视角——AI 改变了哪些架构决策，多 Agent 协作以后系统边界怎么画。

零散教程通常不讲这条主线。

这是我在电子工业出版社出版的畅销书，实体书定价 ¥100。这次把全书重新整理了一遍，把原来受印刷限制删掉的彩色图都加了进去——买过实体书的，也值得重读一遍电子版。

大多数后端工程师都用过 Redis，但真到面试或者生产排障，很多人只会答 set/get、缓存穿透、分布式锁。再往深问：为什么 Redis 快？AOF 和 RDB 怎么选？缓存击穿为什么不是加个锁就完事？Cluster 为什么会有 hash slot？这些问题，书里不只给答案，还给"为什么这样设计"的原理。

**《后端面试高手心法 58 讲》**

很多后端同学不是没能力，是不会讲。项目做过，但讲不出亮点；线上问题排过，但说不成完整路径；Redis、MySQL、Spring 都用过，面试时回答像在背资料。

## 我知道你有几个顾虑

**「我已经在用 ChatGPT 、豆包、DeepSeek 了，还需要学这些吗？」**

ChatGPT、DeepSeek 是"你问它答"，这套专栏讲的是"你给目标，AI 来完成"——读文件、操作软件、自动执行任务、跨软件串联工作流，以及最重要的，把你的工作经验固化成可复用的 AI 行为（Skill）。这是两个不同层次的用法。就像五月这波更新的核心方向——工具从对话框里跑出来，接进你的工作流——学会配置这条路，才是真正的区别。

**「不会代码，真的能跟上非技术那条线吗？」**

AI 超能工具箱 31 讲，没有一讲要求写代码。每一个需要在终端输入的命令，都写出来让你直接复制，并且解释它在做什么。安装步骤有截图，有常见报错解决方法，从没碰过命令行的人也能按步骤走下来。

**「国内环境能用吗？」**

能。AI 编程那套专栏里专门有一讲讲国内可用方案，包括用 cc-switch 接 DeepSeek 的路径，哪些任务可以用国产模型跑，哪些任务仍然建议用更强的模型，怎么在成本和效果之间做取舍。AI 工具那套也考虑了国内可用性，每个工具都标注了可访问情况。

**「星球会不会变成资料收藏夹？」**

我希望你用，不希望你只是加进去。每讲配套有思考题和动手任务，星球里有提问区和实战分享区。你在公司遇到的具体场景，可以拿出来拆，好的问题我会整理进后续加餐或答疑里。林强和李运华在星球里也会参与——这不是买了就消失的那种。

**「你凭什么讲这些？」**

我出版过 Redis 畅销书，在 InfoQ 写了几年文章，在一线大厂做后端架构，是腾讯云架构师同盟名人堂专家。更重要的是，我用了整整一年把 AI 工具真正跑通了自己的工作流，写专栏的内容来自真实踩坑，不是课件搬运。

李运华的资历不需要我多介绍，阿里 P9 在国内技术圈意味着什么大家心里有数。林强在 CodeBuddy 上积累的企业落地经验，在公开渠道很难找到这个密度的第一手内容——这也是我在五月这个时间节点特别想说他的原因：当外面的工具都在往企业场景推进的时候，他已经在那个场景里做了一两年了。
