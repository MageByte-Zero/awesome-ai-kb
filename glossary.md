# AI + 后端技术术语表

## AI 术语

| 术语 | 解释 |
|------|------|
| **LLM**（大语言模型） | 基于 Transformer 架构的大规模语言模型，如 GPT-4、Claude，通过海量文本预训练获得生成与推理能力。 |
| **RAG**（检索增强生成） | 在 LLM 生成回答前，先从外部知识库检索相关文档片段，将检索结果注入提示词以提升回答准确性。 |
| **Agent**（智能体） | 具备自主规划、工具调用、多步推理能力的 AI 系统，能独立完成复杂任务而非单轮问答。 |
| **MCP**（Model Context Protocol） | Anthropic 推出的模型上下文协议，定义 LLM 与外部工具/数据源之间的标准交互接口。 |
| **Embedding**（向量嵌入） | 将文本/图像等非结构化数据映射为高维向量，使语义相近的内容在向量空间中距离更近。 |
| **Chunking**（文本分块） | 将长文档切分为固定大小的文本片段，是 RAG 系统中构建检索索引的前置步骤。 |
| **Prompt Engineering**（提示工程） | 设计、优化输入提示词以引导 LLM 产生期望输出的方法论。 |
| **Tool Use**（工具调用） | LLM 调用外部 API / 函数的能力，使模型能执行搜索、计算、数据库查询等超出纯文本生成的操​​作。 |
| **Fine-tuning**（微调） | 在预训练模型基础上，使用特定领域数据进行额外训练，使模型适配特定任务或风格。 |
| **ReAct** | Reasoning + Acting 模式，LLM 交替进行推理（思考下一步）和行动（调用工具），循环直到任务完成。 |
| **HITL**（Human-in-the-Loop） | 人机协作模式，AI 在关键决策点暂停并等待人类确认或干预。 |

## 后端技术术语

| 术语 | 解释 |
|------|------|
| **Redis** | 开源内存数据结构存储，支持字符串、哈希、列表、集合等类型，广泛用于缓存、消息队列、排行榜。 |
| **Kafka** | 分布式流处理平台，以高吞吐、低延迟、持久化著称，用于日志收集、消息系统、实时数据管道。 |
| **JVM**（Java 虚拟机） | 运行 Java 字节码的虚拟机，负责内存管理（GC）、类加载、JIT 编译，是 Java 生态的基石。 |
| **G1GC**（Garbage-First Garbage Collector） | JVM 的低延迟垃圾回收器，将堆划分为多个 Region，优先回收垃圾最多的区域，适合大堆内存场景。 |
| **MySQL** | 开源关系型数据库，InnoDB 引擎支持事务（ACID）、行级锁、MVCC，是互联网行业主力数据库。 |
| **Druid** | 阿里巴巴开源的数据库连接池，提供 SQL 监控、防火墙、慢查询分析等功能。 |
| **分布式系统** | 由多个独立计算节点组成的系统，通过消息传递协调工作，面临一致性、可用性、分区容错（CAP）的权衡。 |
| **微服务** | 将单体应用拆分为多个独立部署的小服务，每个服务围绕业务能力构建，通过轻量协议（HTTP/RPC）通信。 |
| **Spring Boot** | Java 生态的主流应用框架，通过自动配置和起步依赖简化 Spring 应用的搭建与部署。 |
| **JUC**（java.util.concurrent） | Java 并发编程工具包，提供线程池、锁、原子变量、并发集合等高级并发工具。 |
